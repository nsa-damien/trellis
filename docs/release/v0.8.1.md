# Release v0.8.1

**Release Date:** 2026-01-24

## Highlights

This release refactors the `/trellis.test-plan` command to use a cleaner configuration approach. Instead of `.env` files, tests now use a `settings.json` file with the `--config` flag, providing better separation of concerns and simpler console output.

## What's New

- **Simplified configuration**: Tests use `--config` flag to specify app config path instead of `.env` files
- **Cleaner output**: Console shows PASS/FAIL per test with failure reason only
- **Better logging**: Detailed request/response logs moved to log file for cleaner debugging

## Bug Fixes

No bug fixes in this release.

## Breaking Changes

None. This is a backward-compatible refactoring.

## Upgrade Instructions

1. Update to v0.8.1: `claude plugin update github:NorthShoreAutomation/trellis`
2. If you have existing test configurations using `.env`, migrate to `settings.json`:
   - Create a `settings.json` in your tests directory with your test-specific variables
   - Update test invocations to use `--config path/to/settings.json`
