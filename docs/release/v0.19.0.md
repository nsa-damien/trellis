# Release v0.19.0

**Release Date:** 2026-03-16

## Highlights

New `/trellis:ideas` skill for capturing, tracking, and prioritizing project ideas. Maintains a living `docs/IDEAS.md` backlog with operations for adding, listing, reviewing, and promoting ideas to actionable work. The skill estimates effort levels itself based on project context rather than asking the user.

## What's New

- **`/trellis:ideas`** — Idea tracker with four operations:
  - **add** — Capture ideas inline or interactively, with duplicate detection
  - **list** — Summary view with priority matrix for 10+ active ideas
  - **review** — Interactive prioritization session with re-estimated effort
  - **promote** — Convert ideas to beads issues (or mark promoted without beads)
- **AI effort estimation** — The skill evaluates effort (small/medium/large) based on its understanding of the codebase and project architecture, providing a one-sentence rationale. Users can override if they disagree.

## Upgrade Instructions

No action needed. The new skill is available immediately as `/trellis:ideas`.
