# Release v0.8.0

**Release Date:** 2026-01-20

## Highlights

This release introduces `/trellis.codemap`, a new command that generates semantic code maps for LLM navigation. The command auto-detects project structure across Go, TypeScript/JavaScript, and Python, extracting entry points and key types to create a `CODEMAP.yaml` file that helps LLMs quickly find relevant code.

## What's New

### /trellis.codemap Command

Generate and maintain `CODEMAP.yaml` for LLM code navigation:

- **Auto-detection**: Identifies project type and extracts symbols using language-specific heuristics
- **Hybrid generation**: Auto-detect structure, then guided prompts for semantic enrichment
- **Interactive refinement**: One question at a time for ambiguous modules
- **Update mode**: Detects drift when CODEMAP.yaml exists, validates changes sequentially
- **Spec linking**: Links modules to spec-kit specs when available

**Supported flags:**
- `--dry-run` - Preview without writing
- `--no-interactive` - Skip enrichment questions
- `--language <lang>` - Override detected language
- `--include/exclude <pattern>` - Filter directories
- `--json` - Output as JSON

### /trellis.implement Integration

Implementation sessions now update CODEMAP.yaml automatically:
- Scans modified files for new/changed symbols after all beads complete
- Prompts sequentially for each detected change
- Suggests creating CODEMAP.yaml if none exists

## Upgrade Instructions

1. Update the plugin:
   ```bash
   claude plugin update trellis
   ```

2. (Optional) Generate a code map for your project:
   ```bash
   /trellis.codemap
   ```

No breaking changes in this release.
